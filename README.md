🛡️ PredictaCore Titán: Forensic Audit Intelligence
PredictaCore Titán no es una herramienta de consultoría; es un sistema de auditoría forense simbióptico diseñado para desmantelar activos digitales, identificar fugas de capital y ejecutar protocolos de optimización de conversión mediante IA de última generación (Gemini 2.5 Pro).

🏗️ Arquitectura del Sistema (Mapa de Archivos)
El proyecto está estructurado en capas para proteger la integridad del núcleo mientras se permite la evolución táctica.

1. El Núcleo (Core)
server.js: El centro de mando. Orquesta el flujo desde la captura del activo hasta la generación del reporte. Gestiona la autenticación con Google Vertex AI y la lógica de estados de la auditoría.

motor.js: Los "ojos" del sistema. Utiliza Puppeteer para realizar scraping de alta precisión, capturas de pantalla multimodales (Desktop/Mobile) y simulación de comportamiento humano para evitar bloqueos.

package.json: Registro de dependencias críticas y motores de ejecución.

2. El Cerebro (IA & Prompts)
cerebro.js: Lógica de auditoría web. Contiene los prompts multilingües y las reglas de negocio para sitios e-commerce y corporativos.

cerebro_social.js: Especialización para redes sociales. Filtra el ruido de las plataformas (Meta/TikTok) para auditar exclusivamente el perfil del cliente.

cerebro_tactico.js: (Capa de Élite) Prompts de alta precisión que fuerzan a la IA a usar la visión para señalar errores visuales específicos y realizar benchmarks con datos reales de búsqueda.

firewall.js: El protocolo de identidad. Define las restricciones éticas, el tono agresivo-forense y las prohibiciones de la IA para mantener la autoridad de la marca.

3. La Capa Visual (UI & Reporte)
visual.js: Interfaz de terminal web para el usuario. Es el frontend donde se ingresa el DNA (URL) y se visualiza el progreso en tiempo real.

visual_pro.js: Extensión de estilos para el motor de PDF. Controla la limpieza visual y la estructura de los bloques de datos.

guia_ejecutiva.js: Traductor humano. Añade cápsulas de contexto a cada sección del reporte para eliminar tecnicismos y facilitar la comprensión del cliente.

estilos_titan.js: Capa de maquetación avanzada para el PDF. Sella el desacomodo visual, controla saltos de página y resalta las [CRITICAL HEMORRHAGE].

4. Despliegue (DevOps)
nixpacks.toml: Configuración de entorno para Railway. Asegura que el servidor tenga instaladas todas las librerías de sistema necesarias para que el navegador de Puppeteer funcione en la nube.

🚀 Flujo de Operación
Ingreso de DNA: El usuario proporciona una URL a través de visual.js.

Infiltración: motor.js navega el activo, extrae el dossier de texto (hasta 45,000 caracteres) y genera capturas multimodales.

Procesamiento Forense: server.js envía los datos a Vertex AI, alternando entre los prompts de cerebro.js y las mejoras de cerebro_tactico.js.

Cristalización: El sistema combina los resultados de la IA con la guia_ejecutiva.js y el CSS_TITAN para generar un PDF impecable y letal.

🛠️ Requisitos Técnicos
Node.js: v18.x

Librerías: Express, Puppeteer, Google Auth Library.

Variables de Entorno: GOOGLE_CREDS (JSON de Service Account) y API_KEY.
